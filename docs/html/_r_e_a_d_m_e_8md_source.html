136  # WHERE IN CLAUSE
137  the <b>where_in</b> and <b>where_in_or</b> methods provides means to apply WHERE IN clauses to your table operations
138 
139  ```php
140 //in theaddress table we used above, lets apply some where clauses
141 $l = new LindaModel("address"); //the address table is from the open-source sakila database
142 $l->where_in("city_id", ["300, 400, 500"]); //wher the city_id is within that range
143 
144 ```
145 <b>where_in_or</b> differs significantly, for <b>where_or</b><br/>
146 For where_in_or you are specifying that an OR comes before the where_in CLAUSE, if multiple clauses exixts
147 
148 <br/><br/>so as an example
149 
150 ```php
151 $l = new LindaModel("address"); //the address table is from the open-source sakila database
152 $l->where_in_or("city_id", ["300", "400", "500"]) //wher the city_id is within that range)
153  ->where("city_id","<" ,100); //wher the city_id is within that range)
154 
155 $rows = $l->get()->collection();
156 ```
157 
158 would translate to
159 
160 ```sql
161 SELECT * FROM `address` WHERE( city_id < 100 ) OR city_id IN ('300', '400', '500') LIMIT 0, 1000;
162 ```
163 You'll notice that even though the <b>where_in</b> clause came first, the <b> where clause</b> was processed first<br/>
164 This is a design decision, as thus all WHERE clauses would be processed before WHERE IN's, irrespective of the order the methods where called, you can also pass a sub-query string as the second value of a <b> where_in</b> method call

167 # INNER JOIN CLAUSE
168 Inner Joins are a common way to retrieve related data from multiple table and the <b> LindaModel</b> class provides a convinient method
169  to perform such joins<br/>
170  The INNER JOIn method signature is
171  <b> inner_join($table, $conditional_column_a, $conditional_column_b)</b><br/>
172  where <b>$table</b> is the table you want to JOIN with<br/>
173  <b> $conditional_column_a</b> is column on the current table you are operating on<br/>
174  <b> $conditional_column_b</b> is the column to match on the JOIN'ed table<br/>
175 
176 ```php
177 $l = new LindaModel("address");
178 $l->where("city_id","<" ,100) //wher the city_id is within that range)
179  ->inner_join("city", "city_id", "city_id");
180 
181 
182 $rows = $l->get()->collection();
183 ```
184 In the above an SQL query sililar to
185 ```sql
186 SELECT * FROM `address` AS T1 INNER JOIN `city` AS T2 ON T1.city_id = T2.city_id WHERE( T1.city_id < 100 ) LIMIT 0, 1000;
187 ```
188 would be executed, multiple inner joins are also supported
189 
190 ```php
191 $l = new LindaModel("address");
192 $l->where("city_id","<" ,100) //wher the city_id is within that range)
193  ->where("county_id", ">", 10, 3)
194  ->inner_join("city", "city_id", "city_id");
195  ->inner_join("county", "county_id", "county_id"
196 
197 $rows = $l->get()->collection();
198 ```
199 On the second call to where, the third argument, is the index of the join table the where clause shoule associate with
200 Which means the second where clause should evaluate the column on the third joined table. In a join'ed statement,
201 The base table is taken as the first table i.e T1, the others, T2. , T3. ...
202 ```sql
203 SELECT * FROM `address` AS T1 INNER JOIN `city` AS T2 ON T1.city_id = T2.city_id INNER JOIN `county` AS T3 ON T1.county_id = T3.county_id WHERE( T1.city_id < 100 ) AND ( T3.county_id > 10 ) LIMIT 0, 1000;
204 ```
205 As you can see the second where statement evaluated to T3.county_id, since the county table is the third table in the Join and
206 we specified the join index for the where clause as 3 above

209 # PAGINATION
210 LindaModel supports two methods take() and skip() for paginating reslts
211 ```php
212 $l = new LindaModel("address");
213 $l->where_in_or("city_id", ["300", "400", "500"]) //wher the city_id is within that range)
214  ->where("city_id","<" ,100) //wher the city_id is within that range)
215  ->take(10)
216  ->skip(4);
217 $rows = $l->get()->collection();
218 ```
219 <br/>
220 Would execute the Statement
221 ```sql
222 SELECT * FROM `address` WHERE( city_id < 100 ) OR city_id IN (300,400,500) LIMIT 4, 10;
